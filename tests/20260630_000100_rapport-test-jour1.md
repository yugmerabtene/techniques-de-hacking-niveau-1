## Problèmes identifiés

1. **Bug source path `../../..`** : Tous les scripts `.sh` dans `labs_resolution/jour-01/` utilisent `source "$(...)/../../.."` au lieu de `../..` pour trouver `env.sh`. N'affecte pas l'exécution car les scripts sourcent leurs propres variables.

2. **John v1.9.0** : `--format=raw-md5` non supporté → utiliser hashcat à la place.

3. **DVWA v1.10 CSRF** : Le login nécessite un `user_token` extrait de la page/login.php. Les commandes du cours sans token échouent silencieusement.

4. **DVWA v1.10 "Login failed"** : Le message n'apparaît pas dans la réponse → Hydra ne peut pas distinguer login réussi/échoué.

5. **test-empty/** : Le dossier de test pour le 403 n'existe pas dans cette version → retourne 404.
